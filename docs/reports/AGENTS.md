# docs/reports AGENTS

Generated diagnostic reports that remain active evidence for ongoing work.

These files are not authority. Durable conclusions must be extracted into
manifests, tests, contracts, or authority docs before they become binding.

## File registry

This directory intentionally starts empty. Generated reports are allowed here
only when a declared writer in `architecture/script_manifest.yaml` targets
`docs/reports/`.

## Rules

- Reports here are evidence, not law.
- Tracked report files must be added to this file registry with a stable purpose line.
- Prefer Markdown reports with stable filenames over ad hoc one-off dumps.
- If a report becomes historical, archive it under `docs/archives/reports/`.
- Do not point agents at this directory as a default read surface.
