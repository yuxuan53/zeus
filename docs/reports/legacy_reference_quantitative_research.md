## 第一部分：气象数据源深度研究

### 1.1 结算真相：你到底在交易什么？

在做任何数据源选择之前，必须先回答一个事实问题：**Polymarket 天气市场用什么数据结算？**

根据对当前活跃市场规则的直接检查：

**NYC 温度市场**的结算源不是 NWS、不是 NOAA，而是 **Weather Underground**，具体为 LaGuardia Airport 站点 (KLGA) 的每小时最高温度记录。页面原文明确指定：resolution source 为 `wunderground.com/history/daily/us/ny/new-york-city/KLGA`，精度为整数华氏度。

这个事实产生一个极其重要的后果：**NWS 发布的官方每日最高温度与 Weather Underground 显示的最高温度可以不一致。** 气象学家 Ryan Maue 曾公开指出，NWS 记录 KLGA 日最高为 30°F，但 Polymarket 按 Wunderground 小时观测结算为 29°F。差别原因：NWS 的"每日最高"基于气候日（通常 midnight-to-midnight 或 morning-to-morning），而 Wunderground 显示的是逐小时记录中的最大值，两者的时间窗口可能不同。

**London 温度市场**同样使用 Weather Underground，典型站点为 London City Airport (EGLC)，精度为整数摄氏度。

**NYC 降水市场**使用 NOAA 月度降水数据，观测站为 Central Park，精度 0.01 英寸。

**全球温度异常市场**使用 NASA GISS 全球陆地-海洋温度指数。

**核心原则：你的预测目标不是"明天 NYC 多少度"，而是"Weather Underground KLGA 页面明天会显示的最高温度整数值是多少"。** 这两个问题的答案通常相同，但在边界情况下可以不同，而边界情况恰恰是你最容易有 edge 的地方。

### 1.2 数值天气预报模型（NWP）

#### GFS（全球预报系统）

GFS 是美国 NOAA/NCEP 运行的旗舰全球模型。

**更新频率**：每天 4 次（00Z, 06Z, 12Z, 18Z），每次运行后约 3.5-4 小时开始出数据。预报时效覆盖 0-384 小时（16 天），前 120 小时为逐小时输出，之后为每 3 小时。

**空间分辨率**：0.25°（约 28km），这意味着 NYC LaGuardia 的 2m 温度预报实际上是 LaGuardia 所在网格点（约 28km × 28km）的面平均值，而非站点值。这个差异在沿海城市尤为显著——网格可能包含水体，拉低夏季高温预报。

**时效精度衰减**（2m 温度 RMSE，北半球中纬度，典型值）：
- T+24h: ~1.5°C（~2.7°F）
- T+48h: ~2.0°C（~3.6°F）
- T+72h: ~2.5°C（~4.5°F）
- T+120h: ~3.5°C（~6.3°F）
- T+168h: ~4.5°C（~8.1°F）

**免费获取方式**：NOMADS (`nomads.ncep.noaa.gov`)，提供 GRIB2 格式原始数据。解析 GRIB2 需要 `cfgrib` 或 `eccodes` 库，工程成本不低。

**已知偏差**：GFS 在美国东部城市有季节性系统偏差——夏季偏暖约 1-2°F（过度估计日间加热），冬季在强冷锋后偏冷约 1-3°F（低估边界层恢复速度）。

#### ECMWF（IFS 高分辨率确定性预报 + ENS 集合预报）

ECMWF 是全球公认最准确的中期预报模型。根据 DIMOSIC 项目（不同模型、相同初始条件）的系统性验证，IFS 在 1-3 天时效内显著优于所有其他模型。在 5 天以上时效，ICON（德国气象局）与 IFS 接近，但 IFS 仍保持统计显著优势。

**关键数据：ECMWF 保持着约一天的精度领先**——即 ECMWF 的 6 天预报与 GFS 的 5 天预报精度相当。对于交易来说，这意味着在 T+5 到 T+7 的时效窗口，ECMWF 的信号比 GFS 有显著的信息优势。

**ENS 51 成员集合预报的概率转换**：

ENS 包含 1 个控制运行（未扰动初始条件）和 50 个扰动运行。每个成员代表初始条件和模型物理的一种可能实现。将其转换为阈值概率的基本方法：

```
P_raw(T_max > threshold) = count(member_max_temp > threshold) / 51
```

但这个原始概率有两个已知问题：

1. **欠离散性（underdispersion）**：ECMWF ENS 的集合离散度系统性偏小，导致极端事件的概率被低估，近中值事件的概率被高估。具体表现为：ENS 给出 P=0.90 的事件实际发生率约 0.85，给出 P=0.10 的事件实际发生率约 0.15。
2. **2m 温度的分辨率问题**：ENS 水平分辨率为 9km（2024 年升级后），仍然太粗以准确捕捉城市热岛、海岸效应、和局地地形效应。

**获取方式**：Open-Meteo API 免费提供 ENS 数据（见 1.2.4）。直接从 ECMWF 获取需要付费或使用 Copernicus Climate Data Store（CDS），后者免费但数据延迟约 5 天，不适合实时交易。

#### HRRR（高分辨率快速刷新）

**更新频率**：每小时一次。空间分辨率 3km。预报时效 0-18h（每小时循环）或 0-48h（00Z/12Z 循环）。

**核心价值**：HRRR 每小时同化一次雷达数据，这意味着它能捕捉到其他模型完全看不到的短时天气演变。对于当天或次日的温度预报，HRRR 在美国大陆上几乎不可替代。

**精度对比**：在 T+6h 到 T+18h 时效内，HRRR 的 2m 温度 RMSE 约 1.0-1.5°C，显著优于 GFS 的 1.5-2.0°C。在 T+1h 到 T+6h 的极短时效，HRRR 的优势更大。

**限制**：仅覆盖北美（CONUS + 部分加拿大南部）。不适用于 London、Seoul、Shanghai 等市场。

**获取方式**：AWS Open Data (`s3://noaa-hrrr-bdp-pds`)，GRIB2 格式，免费。或通过 Herbie Python 库方便提取。

#### NAM（北美中尺度模型）

**更新频率**：每天 4 次。分辨率 12km（主域）和 3km（嵌套域）。预报时效 0-84h。

**定位**：NAM 介于 GFS（全球、粗分辨率、长时效）和 HRRR（区域、高分辨率、短时效）之间。在 T+24h 到 T+60h 的窗口内，NAM 的 3km 嵌套域对冷空气堆积（cold air damming）、阿巴拉契亚地形效应、和大湖效应降雪有特殊优势。

**交易价值判断**：对于 NYC 温度市场，NAM 可以作为 GFS 的 cross-check，但不值得作为独立信号源投入工程成本。

#### Open-Meteo

Open-Meteo 是一个免费 API，聚合了多个底层 NWP 模型的后处理输出。

**底层模型包括**：ECMWF IFS、GFS、ICON（德国）、GEM（加拿大）、JMA（日本）、和多个区域模型（MétéoFrance AROME/ARPEGE、KNMI HARMONIE 等）。集合预报 API 提供 ECMWF ENS 51 成员数据。

**数据延迟**：Open-Meteo 在模型运行完成后 1-3 小时内更新数据。对于 ECMWF 00Z 运行，数据通常在约 08:00-09:00 UTC 可用。GFS 延迟略短。

**优势**：
- 完全免费，无需 API key（商业用途每天 10,000 次调用限制）
- 统一 JSON API，无需解析 GRIB2
- 提供 MOS（模型输出统计）后处理的温度预报，已经做了部分偏差校正
- 集合预报 API 直接返回 51 个成员的逐小时温度

**劣势**：
- 你无法控制后处理算法——当 Open-Meteo 更新其 MOS 方法时，你的校准曲线可能失效，且你不会收到通知
- 数据延迟意味着在模型运行后 1-3 小时内，直接访问 NOMADS 的交易者有信息优势
- 无法获取原始模型输出的完整变量集（如 850hPa 温度、边界层高度等诊断变量），这些变量对理解预报"为什么"偏了很有价值

**关键延迟风险**：对于 "T+0"（当天）市场，ECMWF 00Z 运行的 Open-Meteo 数据在约 09:00 UTC 可用。如果市场在 23:59 UTC 结算，这给你约 15 小时的交易窗口。但如果 Open-Meteo 服务器延迟（历史上有过 6-8 小时延迟的记录），你的交易窗口可能缩短到不足以执行。

### 1.3 观测数据

#### Weather Underground 历史观测

**这是结算数据源，因此也是最重要的"观测"数据。**

Weather Underground 从 ASOS/AWOS 自动观测站获取数据，但其网页显示的值经过自己的处理：
- 温度显示为整数（四舍五入）
- "Daily High" 是当天所有小时观测中的最大值
- 数据可能在结算后被"finalized"，即可能有短期延迟或修正

**关键偏差**：Weather Underground 的 "highest temperature" 是基于该站点所有时间观测的最大整数温度。这与 NWS 的 "official daily maximum temperature"（基于气候日定义）可能不同。尤其在以下场景：
- 午夜前后温度变化：如果 23:50 LST 温度为 31°F，00:10 LST 温度为 30°F，Wunderground 可能将两天都记为不同值
- 冷锋通过时：温度可能在清晨达到当天最高，之后持续下降
- 夏季午后雷暴：暴风外流（outflow boundary）可能在 30 分钟内降温 10-15°F

#### ASOS 站点数据（IEM）

Iowa Environmental Mesonet (IEM) 提供所有 ASOS 站点的历史 1 分钟和 5 分钟数据下载，URL 格式已知且稳定。

**用途**：构建 KLGA、EGLC 等结算站点的历史温度分布，用于校准曲线训练。

**与 Wunderground 的系统偏差**：IEM 提供的是原始 METAR/SPECI 报文解析值，精度为 0.1°C（然后转换为整数°F）。Wunderground 的处理可能使用不同的四舍五入规则或时间窗口。在构建校准数据集时，应直接使用 Wunderground 的历史值作为 ground truth，而非 IEM 的原始观测。

#### NOAA GHCN-Daily

全球历史气候网络每日数据，提供数十年的日最高/最低温度记录。

**用途**：构建气候基准（climatology）。例如，"NYC 3月29日的历史日最高温度均值为 54°F，标准差约 8°F"。这个基准用于判断当前预报是否异常，以及市场定价是否合理锚定在气候学上。

**已知偏差**：GHCN-Daily 存在"观测时间偏差"（time-of-observation bias）——不同站点在不同时间读取每日极值，导致系统性地高估或低估日最高/最低温度。对于 KLGA（机场站点），观测时间是标准的 midnight-to-midnight，偏差较小。

### 1.4 集合预报的特殊价值

#### 为什么单点预报不够？

因为 Polymarket 温度市场的支付结构（payoff structure）是**非线性的**。

考虑一个市场："NYC 明天最高温度 52-53°F？" 假设 GFS 确定性预报给出 53°F。这个信息不够——你需要知道预报是 53°F ± 1°F（高置信度，P(52-53°F) 很高）还是 53°F ± 5°F（低置信度，P(52-53°F) 较低）。

单点预报丢失了不确定性信息。集合预报直接给你不确定性的度量。

#### 集合离散度作为不确定性度量

ECMWF ENS 的集合离散度（ensemble spread，通常用标准差度量）与预报误差正相关，但关系不是线性的：

- **小离散度（< 1.5°C）**：模型对大气状态有高共识，预报误差通常也较小。但注意 ECMWF ENS 是欠离散的——即使离散度为 1°C，实际误差可能是 1.5°C。
- **大离散度（> 3°C）**：模型成员之间分歧严重，通常意味着天气系统处于转折点（regime change），如冷锋通过时间不确定、或强对流是否影响站点不确定。此时预报极不可靠。
- **双峰分布**：偶尔 ENS 成员会聚类为两组（如 30 个成员预报 40°F 附近，21 个成员预报 50°F 附近），这通常意味着预报依赖于某个二元事件（冷锋是否在日落前通过）。此时中值是无意义的，必须分析两种情景的概率。

#### 模型共识度与市场隐含概率

一个实用的度量是"**模型共识度**"（model agreement ratio）：

```
对于 Polymarket 的一个温度区间 [a, b]：
model_consensus = count(member_max ∈ [a, b]) / 51
market_implied_p = market_price_yes
```

当 `model_consensus - market_implied_p > threshold` 时，可能存在交易机会。但"threshold"的正确值取决于以下校准问题。

## 第二部分：核心数学体系

### 2.1 概率校准体系

#### 问题定义

你有一个"原始概率"——ENS 给出的 `P_raw = count(members > threshold) / 51`。你需要将它校准为"真实概率"——即与实际结果频率一致的概率。

#### 选择：Platt Scaling

**在样本量 < 500 的场景下，我选择 Platt Scaling（逻辑回归校准），而非 Isotonic Regression 或 Beta Calibration。**

理由：

**Isotonic Regression** 是非参数的，自由度高，理论上能拟合任意单调校准曲线。但 scikit-learn 文档和 Niculescu-Mizil & Caruana (2005) 的经典论文都明确指出：**isotonic regression 在样本量 < 1000 时容易过拟合**。你的训练集是每个城市每个预报时效的历史结算记录——即使收集两年数据，每个 (城市, 时效, 季节) 组合的样本量也不太可能超过 200。Isotonic regression 在这个样本量下会产生阶梯状的校准曲线，在数据稀疏的尾部区域（P < 0.1 或 P > 0.9）尤其不可靠——而尾部恰恰是你最需要准确校准的地方。

**Beta Calibration** 是一个不错的中间方案（3 个参数 vs Platt 的 2 个），能拟合比 sigmoid 更灵活的曲线。但在样本量 < 300 时，多出的参数带来的拟合灵活性不足以抵消过拟合风险。

**Platt Scaling** 只有 2 个参数（A, B），对应一个 sigmoid 变换：`P_cal = 1 / (1 + exp(A * P_raw + B))`。它假设校准曲线是 S 形的——这对 ECMWF ENS 的欠离散性来说是一个合理的假设（ENS 的概率系统性地向 0.5 偏移）。

#### 校准曲线需要多少样本？

这是一个核心问题。答案取决于你需要的精度。

考虑一个简单的 frequentist 估计：你有 N 个预报-结果对，其中 k 个预报的 P_raw 落在 [0.4, 0.6] 的 bin 中，实际命中 m 个。命中率的 95% 置信区间宽度约为：

```
CI_width ≈ 2 * 1.96 * sqrt(p*(1-p) / k)
```

对于 p = 0.5, k = 50，CI_width ≈ 0.28。即你无法区分 0.36 和 0.64 的真实概率。对于 k = 200，CI_width ≈ 0.14，稍好但仍然粗糙。

**实际意义**：在你的 edge 通常只有 5-10% 的场景下，你需要每个校准 bin 至少 200-300 个样本，才能有意义地校准。在样本不足的初期，正确的做法是：

**降级策略**：

| 样本量 | 校准方法 | 置信度处理 |
|--------|----------|-----------|
| < 30 per bin | 不做校准，使用 P_raw + 扩大不确定性 2x | 只交易 edge > 15% 的极端情况 |
| 30-100 per bin | Platt Scaling，但将 edge 阈值提高到 10% | 保守仓位 |
| 100-300 per bin | Platt Scaling | 标准 edge 阈值 5% |
| > 300 per bin | 可尝试 Beta Calibration 或 Isotonic | 标准操作 |

#### 概念漂移检测（Calibration Drift）

如何检测校准模型已经失效？使用 **Hosmer-Lemeshow 检验的滚动版本**：

将最近 N 笔交易（N ≥ 50）按校准后概率分为 5 个 bin，对每个 bin 计算 (预期命中数 - 实际命中数)² / 预期命中数。如果总 chi-squared 统计量超过临界值（5 个 bin 对应自由度 = 3，α=0.05 时临界值 7.81），标记校准可能失效。

**更简单的启发式方法**：如果连续 20 笔交易中，有 8 笔以上的校准概率与实际结果方向相反（即校准概率 > 0.6 但结果为 NO，或校准概率 < 0.4 但结果为 YES），触发重新校准。

### 2.2 市场定价错误的识别

#### Edge 是真的还是模型误差？

你的模型说 P = 0.65，市场价格隐含 P = 0.55。这 10% 的差距有多少是真 edge？

关键认知：**你不知道**，你只能估计。这里需要的不是一个点估计，而是一个关于 edge 的分布。

#### 选择：贝叶斯融合

**市场价格本身包含信息。** Polymarket 的天气市场参与者包括使用类似数据源的 bot（gopfan2, meropi 等已证明这点），也包括有独立信息的交易者。将市场价格视为一个与你的模型独立的概率估计，进行贝叶斯融合：

```
P_posterior ∝ P_model^w_model * P_market^w_market
```

但这个公式假设两个估计是独立的——它们不是（都基于相同的天气模型）。更合理的框架：

```
P_posterior = α * P_model + (1-α) * P_market
```

其中 α 取决于你对自己模型的相对信心。初始阶段 α = 0.5（即你的模型信号和市场信号权重相等），随着校准数据积累和 edge 验证，逐步调高。

**Edge 的置信区间**：

使用 Bootstrap 方法。从你的校准训练集中有放回抽样 1000 次，每次重新拟合 Platt Scaling，得到 1000 个 P_cal 值。你的 edge 分布是：

```
edge_distribution = [P_cal_i - P_market for i in 1..1000]
```

只有当 edge 分布的 5th percentile > 0 时（即 95% 置信区间下界为正），才认为 edge 是统计显著的。

**为什么不用频率学派的 z-test？** 因为 z-test 假设 P_cal 的误差是正态分布的，这在尾部概率（P < 0.1 或 P > 0.9）附近不成立。Bootstrap 不需要分布假设。

### 2.3 仓位管理的数学基础

#### 预测市场中 Kelly 准则的正确形式

传统 Kelly：`f* = (p*b - q) / b`，其中 p = 胜率，b = 赔率，q = 1-p。

在 Polymarket 二元市场中，你以价格 c 买入 YES share。如果赢，你得到 $1（净赚 1-c）。如果输，你亏 c。因此：

```
b = (1-c) / c    （赔率 = 赢利/亏损）
p = P_model      （你认为的真实概率）
q = 1 - P_model

f* = (P_model * (1-c)/c - (1-P_model)) / ((1-c)/c)
   = (P_model * (1-c) - (1-P_model) * c) / (1-c)
   = (P_model - c) / (1-c)
```

这个公式极其简洁：**Kelly 下注比例 = edge / 赢时利润率**。如果你认为 P = 0.65 且价格 c = 0.55，则 f* = (0.65-0.55)/(1-0.55) = 0.222，即下注 22.2% 的资金。

但这是 Full Kelly——**绝不能使用 Full Kelly。**

#### 为什么 Quarter Kelly？

Full Kelly 假设：
1. 你的概率估计 P_model 是准确的——它不是
2. 市场可以无限流动性地执行你的订单——它不能
3. 你不关心单次大幅亏损——你关心

Quarter Kelly（f = f*/4）的理论依据来自 Kelly 本人的论文和后续研究：

- 使用 half Kelly 时，预期增长率是 full Kelly 的 75%，但方差减少约 50%
- 使用 quarter Kelly 时，预期增长率是 full Kelly 的 ~56%，但最大回撤的中位数下降约 75%
- 对于估计误差 σ_p 的 edge，最优 Kelly 分数接近 `f* / (1 + σ_p² / edge²)`。当你的 edge 估计不确定性与 edge 本身大小相当时（σ_p ≈ edge，这在你的场景中很可能），最优分数约为 f*/2

**我的选择**：默认 Quarter Kelly，在高置信度场景（Bootstrap 95% CI 下界 > 5%，且样本量 > 100）可升级到 Third Kelly。

#### 多市场同时交易的 Kelly 扩展

当你同时在 k 个市场持仓时，简单地对每个市场独立计算 Kelly 是错误的——因为市场之间可能相关。

**正确方法**：联合 Kelly 需要求解一个凸优化问题，最大化 `E[log(W)]` 其中 W 是所有市场结果的联合函数。对于二元独立市场，这简化为独立 Kelly 的总和受约束于总仓位 < 1。但天气市场不独立——同一城市的相邻日期温度高度相关，同一天不同城市可能因同一天气系统而相关。

**实用近似**：
1. 对每个市场独立计算 Quarter Kelly
2. 将相关市场的仓位合计视为单一风险暴露
3. 总仓位上限 50% 的资金
4. 同一地理区域的市场仓位上限 25%

这不是数学最优的，但避免了估计协方差矩阵（你的样本量根本不够用来可靠估计）。

#### 最大回撤约束

Kelly 框架本身不约束最大回撤。添加约束的方式：

将 Kelly 分数 f 乘以一个调整因子 `min(1, (max_drawdown_tolerance - current_drawdown) / max_drawdown_tolerance)`。当 current_drawdown 接近 max_drawdown_tolerance 时，仓位自动缩减到零。

这等价于一个线性仓位缩减规则，在 drawdown 达到容忍度时完全停止交易。简单但有效。

### 2.4 预测融合体系

#### 多模型融合方法选择

你有多个信号源：ECMWF ENS, GFS, HRRR（如果适用），历史气候基准。如何融合？

**选择：简单等权平均（初期）→ 基于近期表现的指数加权平均（成熟期）**

理由链：

1. **Bayesian Model Averaging (BMA)** 是理论最优，但需要为每个模型估计一个后验权重，这在小样本下极不稳定。BMA 在气象学中主要用于有数千个验证案例的国家级预报服务。你的样本量不支持。

2. **Stacking（元学习）** 训练一个二级模型将多个预报组合。同样需要大量训练数据，且容易过拟合。

3. **加权平均** 需要可靠地估计各模型的权重。你的 LOO 交叉验证已经确认动态权重在当前样本量下不优于等权——这是正确的结果，不要与之对抗。

4. **简单等权平均** 在小样本下出人意料地强大。这被称为"预测民主"（forecast democracy）或"模型平均的 1/N 规则"，在投资组合理论中有类似的结果（DeMiguel et al. 2009 证明 1/N 投资组合在样本量 < 25×N 时优于均值-方差优化）。

**升级路径**：当累积了 500+ 验证样本后，切换到指数加权平均：

```
w_model_t = w_model_{t-1} * exp(-η * loss_t)
w_model_t = w_model_t / sum(all_weights)  # 归一化
```

η（学习率）设为 0.05-0.1。这种方式自动降低近期表现差的模型的权重，但不会完全丢弃它（避免在模型暂时表现差但未失效时过度反应）。

#### 不同预报时效的融合

"时效越短权重越高"这个简单规则**基本正确，但需要修正**。

正确的处理方式是将不同时效的预报视为对同一事件的不同精度观测。如果你在 T-72h、T-48h、T-24h 分别获得了预报更新：

```
P_final = w_72 * P_72 + w_48 * P_48 + w_24 * P_24
```

其中 w 正比于 1/σ²（预报误差方差的倒数）。由于 σ 随时效增加近似线性增长：

```
σ_24 ≈ 1.5°C, σ_48 ≈ 2.0°C, σ_72 ≈ 2.5°C
```

则 w_24 : w_48 : w_72 ≈ 1/2.25 : 1/4.0 : 1/6.25 ≈ 0.50 : 0.28 : 0.18（归一化后 0.52 : 0.29 : 0.19）。

**关键例外**：当 T-24h 预报与 T-72h 预报发生**方向性冲突**时（如 T-72h 说最高 50°F，T-24h 说最高 38°F），不要简单平均。这通常意味着天气系统发生了根本性改变（如冷锋提前到达），此时应以 T-24h 为准并大幅降低对旧预报的权重。

## 第三部分：已知失效模式

### 3.1 数据层失效

#### 气象模型的系统性温度偏差

GFS 和 ECMWF 都有已知的、与季节和地区相关的 2m 温度系统偏差：

**GFS 已知偏差**：
- 夏季（JJA）在美国东部城市偏暖 1-2°F，原因是过度估计短波辐射加热
- 冬季（DJF）强冷锋后偏冷 1-3°F，原因是边界层参数化不足
- 春秋季转换期（MAM/SON）偏差最小，这也是 GFS 最有交易价值的季节
- 近水体站点（如 KLGA 紧邻 Long Island Sound）全年偏暖约 1°F，因为模型网格的水体占比与实际不符

**ECMWF 已知偏差**：
- ENS 全年有欠离散偏差（ensemble underdispersion）：给出的概率分布比实际窄约 15-20%
- 2024 年 Cycle 48r1 升级后，ENS 分辨率从 18km 提升到 9km，2m 温度偏差有改善，但仍存在
- 强辐射冷却条件下（晴朗冬夜），ECMWF 系统性高估夜间最低温度，因为稳定边界层参数化是全球 NWP 的公认难题

**校正方法**：对于城市级别的温度预报，最有效的校正不是物理校正，而是统计校正。针对 KLGA 站点，建立过去 365 天的 (模型预报, Wunderground 实际值) 对，拟合一个简单的线性校正：

```
T_corrected = a * T_model + b
```

按季节分别拟合（4 组参数）。这消除了大部分系统偏差。线性模型足够——非线性校正在这个样本量下过拟合。

#### ASOS 仪器故障与数据缺失

ASOS 站点可能因以下原因出现数据缺失或异常：
- 温度传感器结冰（冬季）
- 通信中断（断网、断电）
- 站点维护（定期校准）

**对结算的影响**：Polymarket 规则通常规定"data is finalized"后结算，这意味着如果 Wunderground 在结算时显示的值是来自部分缺失数据的，市场将以该值结算。如果整日数据缺失，Polymarket 通常会延迟结算或使用替代源。

**交易策略影响**：当你检测到站点数据质量异常时（如连续多小时温度不变，或温度突然跳变 > 10°F），应视为结算风险事件，减小或清空该市场的仓位。

#### Open-Meteo 数据更新延迟

Open-Meteo 的正常延迟为 1-3 小时，但在以下情况可能更长：
- ECMWF 模型运行延迟（偶尔发生）
- Open-Meteo 服务器过载（通常在极端天气事件期间）
- API 限流（商业用户每天 10,000 次）

**信息盲区风险**：对于当天结算的市场，如果你的数据更新在 12:00 UTC，而市场在 23:59 UTC 结算，你有 12 小时没有模型更新。在这 12 小时内，实际天气可能已经显著偏离预报——尤其是当强对流活动影响区域时。

**缓解**：直接监控 Wunderground 实时温度。当天市场中，最有价值的信号不是模型预报（因为你已经可以直接看到温度在演变），而是判断"当前温度趋势是否会在剩余时间内改变"。HRRR 的每小时更新在这个场景下非常有价值。

### 3.2 模型层失效

#### 气候体制突变

当 ENSO（El Niño/La Niña）、北极涡旋、MJO（Madden-Julian Oscillation）等大尺度气候模式发生突变时，基于近期历史训练的校准模型可能系统性失效。

**举例**：2024-2025 冬季，El Niño 向 La Niña 的快速转换导致美国东部冬季温度模式与前几年显著不同。基于 2022-2024 数据训练的校准曲线可能系统性地高估 NYC 冬季温度。

**早期预警系统**：
1. 监控 CPC（Climate Prediction Center）的 ENSO 月报和季节展望
2. 当 ENSO 状态发生类别变化（El Niño → Neutral → La Niña）时，标记所有校准曲线为"可疑"
3. 在状态转换期间，将 edge 阈值提高到 2 倍
4. 北极涡旋崩溃事件（Sudden Stratospheric Warming, SSW）可以提前 10-14 天从平流层预报中检测到。SSW 后 2-4 周，中纬度温度模式可能发生根本性改变——此时暂停所有基于历史校准的交易。

#### 过拟合 vs 真实 Edge 的区分

这是最困难的统计问题。以下是可操作的检验：

**样本外验证**：将数据按时间分为训练集（前 70%）和测试集（后 30%）。如果模型在训练集上的 Brier score 显著优于测试集（差异 > 0.05），这是过拟合的信号。

**排列检验（Permutation Test）**：将你的交易结果序列随机打乱 1000 次，每次计算盈利。如果你的实际盈利排在打乱结果的 95th percentile 以上，你的 edge 有统计显著性。如果不是——你可能只是运气好。

**最诚实的问题**：在 n = 100 笔交易中，你的胜率是 57%。这与 50%（无 edge）有统计显著差异吗？

```
z = (0.57 - 0.50) / sqrt(0.50 * 0.50 / 100) = 1.4
p-value = 0.16 (双侧)
```

答案是**没有**。你需要约 200 笔交易才能在 5% 显著水平下区分 57% 和 50%。需要约 800 笔交易才能区分 54% 和 50%。

**这是整个系统的核心困境**：在你积累到足够样本量证明 edge 存在之前，你必须基于"我可能有 edge 但无法统计证明"来交易。解决方法不是更复杂的统计——而是**小仓位、多交易、长期积累。**

#### 小样本幻觉

以下统计结论在你的样本量下**不可信**：

| 声明 | 需要最少样本量 | 你可能有的样本量 |
|------|---------------|----------------|
| "ECMWF 比 GFS 更准" | ~300 对比对 | ~50-100 |
| "夏季 edge 更大" | ~200 per 季节 | ~30-50 |
| "我的校准将 Brier 从 0.25 降到 0.20" | ~500 | ~100-200 |
| "动态权重优于等权" | ~800 | ~200 |
| "我的胜率是 55%±5%" | ~400 | ~100 |

**可信的结论**（即使在 n=50 下）：
- 模型预报的方向性正确率是否明显高于基于气候学的基准？（如果一半以上的预报落在正确的 2°F bin 中，而气候学只给出 ~15-20% 的 bin 命中率，这是强信号）
- 是否存在明显的系统偏差？（如果 50 笔交易中模型系统性地高估或低估温度 > 2°F，这在 n=50 下是统计显著的）

### 3.3 市场层失效

#### 流动性限制

Polymarket 天气市场的典型流动性：
- NYC 当日/次日温度：总成交量 $100K-500K，瞬时 order book depth ~$500-2,000 per price level
- NYC 3 天后温度：总成交量 $30K-100K，order book 更薄
- London/Seoul/Shanghai：总成交量 $50K-300K，流动性波动更大

**实际约束**：一笔 $200 的市价单可能导致 2-5% 的滑点。一笔 $500 的市价单可能导致 5-10% 的滑点。这意味着 Kelly 建议的仓位大小常常不可执行——你的有效 edge 被执行成本吃掉。

**解决方案**：在计算 edge 时，将预期滑点作为交易成本扣除。如果你的 edge 是 5%，预期滑点是 3%，你的有效 edge 只有 2%。如果有效 edge < 你的最小阈值，不交易。

#### 鲸鱼效应

大额交易者可以在短时间内将价格推移 10-20%。这会被你的系统错误识别为 edge——"模型说 65%，市场突然变成 45%，edge = 20%"。但这个 "edge" 可能只是流动性冲击，价格很快会回归。

**检测方法**：
1. 如果某个市场的价格在 < 30 分钟内变化 > 10%，标记为"鲸鱼活动"
2. 在鲸鱼活动后 2 小时内不下单——等待价格稳定
3. 如果 2 小时后价格回归到变化前水平 ± 3%，确认为鲸鱼冲击；如果没有回归，可能是新信息到达，需要重新评估

#### 结算前最后几小时的价格行为

在市场结算前 6-12 小时，价格行为发生质变：

1. **信息已知或近乎已知**：对于当天温度市场，下午 3-4 点后日最高温度可能已经出现，温度开始下降。此时正确 bin 的价格应接近 $1，其他 bin 应接近 $0。
2. **套利机会**：如果所有 bin 的 YES 价格之和 ≠ $1（通常是 $0.95-1.05），存在无风险套利空间（卖出所有高价 bin 的 NO）。但这个空间通常很小且被 bot 快速吃掉。
3. **结算风险**：Wunderground 可能在结算后修正数据。如果你在正确 bin 买入 $0.95，但结算后数据被修正导致另一个 bin 胜出，你亏损 $0.95。

**策略调整**：
- 不在结算前 4 小时开新仓
- 在结算前 2 小时，如果你的仓位利润 > 80% 的最大可能利润，考虑平仓锁定利润而非持有到结算

## 附录：数学体系选择总结

| 模块 | 选择 | 核心理由 | 失效条件 |
|------|------|----------|----------|
| 概率校准 | Platt Scaling (2 参数) | 样本量 < 500 时最不易过拟合 | 校准曲线非 sigmoid 形（如 U 形） |
| Edge 识别 | Bootstrap 置信区间 | 不需要正态假设 | Bootstrap 样本本身太小 (< 30) |
| 市场融合 | 线性融合 α*model + (1-α)*market | 简单且不需要独立性假设 | α 的最优值随市场成熟度变化 |
| 仓位大小 | Quarter Kelly | 对估计误差鲁棒 | edge 估计误差 > edge 本身 |
| 多模型融合 | 等权平均（初期）→ 指数加权（成熟期） | 小样本下 1/N 规则最优 | 某个模型系统性失效但仍占 1/N 权重 |
| 时效融合 | 反方差加权 | 有理论基础（最小方差线性无偏估计） | 时效间预报方向性冲突 |
| 回撤控制 | 线性仓位缩减 | 简单、可预测 | 连续小亏损不触发但总回撤很大 |

每一个选择都偏向简单和鲁棒，牺牲了理论最优性。这是有意的——在你的资源和样本量约束下，复杂方法的过拟合风险大于它带来的精度提升。当你积累了 1000+ 验证样本后，重新评估所有选择。
