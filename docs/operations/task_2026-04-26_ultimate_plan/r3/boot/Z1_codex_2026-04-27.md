- **Authoritative truth surface here:** live venue side effects must be gated
  before executor command persistence or SDK calls. Z1 owns only the cutover
  runtime-control state; venue command truth remains owned by
  `src/state/venue_command_repo.py`.
- **Zones touched:** K1 control (`src/control`), K2 execution
  (`src/execution/executor.py`), K2 orchestration
  (`src/engine/cycle_runner.py`), tests, and machine registries.
- **Invariant scope:** INV-NEW-A plus existing INV-25, INV-26, INV-28/30/31,
  NC direct-venue-command constraints, and the live-money contract claim that
  no live placement proceeds unless CutoverGuard is `LIVE_ENABLED`.
- **Operator authority scope:** transitions require an HMAC-signed operator
  token (`ZEUS_CUTOVER_OPERATOR_TOKEN_SECRET`) and `LIVE_ENABLED` additionally
  requires a concrete evidence artifact path. Unsigned ambient strings must not
  move the state machine.
- **Allowed files for implementation:** `src/control/cutover_guard.py`,
  `src/execution/executor.py`, `src/engine/cycle_runner.py`,
  `tests/test_cutover_guard.py`, and required registry/packet files.
- **Forbidden / deliberately deferred:** no live cutover, no real SDK calls in
  tests, no heartbeat cadence claims (Z3), no pUSD/FX classification (Z4/Q-FX),
  no exchange reconciliation table semantics (M5), no direct cancel/redeem
  side-effect rewiring (M4/R1), and no lifecycle enum edits.
- **Change class:** architecture/control feature with live-money guard rails.
- **Semantic boot profile:** none of the source/settlement/Day0/calibration
  profiles directly owns cutover-control work. Fatal misread still applies:
  graph/topology output is routing evidence, not semantic authority.
