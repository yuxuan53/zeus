## Routing proof questions

- **Authoritative truth surface here:** Z2 owns the adapter contract and
  envelope provenance. It does not own venue command schema, lifecycle grammar,
  pUSD accounting, heartbeat policy, or cutover-wipe reconciliation.
- **Zones touched:** K0 contract (`VenueSubmissionEnvelope`), K2 venue/data/
  execution seams, tests, and registry/docs topology.
- **Invariant scope:** NC-NEW-G (envelope provenance, not SDK shape),
  INV-NEW-B (submit produces envelope before side effect), plus existing
  INV-24/25/28/30 live venue ordering.
- **Operator gate:** Q1-zeus-egress is OPEN. Engineering may implement the
  adapter and tests, but `preflight()` must fail closed without the operator
  host-probe artifact.
- **External SDK ambiguity gate:** official docs and wheel source disagree on
  constructor shape. Z2 follows the wheel source for Python (`py_clob_client_v2`)
  and records the discrepancy in the reference excerpt; no live cutover is
  attempted.
- **Deep-interview ambiguity gate:** no user-blocking ambiguity remains for the
  first implementation pass because the phase card plus R3 protocols define the
  non-goals. The main clarified non-goal is: do not wire U1/U2 schema columns or
  M4/R1 cancel/redeem semantics in Z2.
