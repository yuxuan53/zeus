## Stable semantics

- Executable market snapshots are append-only evidence; update/delete is
  blocked by SQLite triggers.
- Snapshot freshness is enforced in Python at
  `venue_command_repo.insert_command()` because the check depends on `now()`.
- New venue commands must cite `snapshot_id` before insertion.
- Missing, unavailable, stale, inactive, closed, orderbook-disabled, or
  `accepting_orders=False` snapshots fail closed.
- Command token id must match the snapshot YES/NO token map and selected token
  when the snapshot has post-decision selection.
- Submitted price must align to `min_tick_size`, submitted size must be at
  least `min_order_size`, and provided `neg_risk` must match the snapshot.
- Executor entry and exit paths pass executable snapshot fields into the
  command insertion gate before V2 preflight, signing, SDK submit, or live
  compatibility wrapper contact.
- Legacy live compatibility paths without a snapshot reject before SDK contact;
  idempotency lookups may return existing commands without re-submitting.
- Fresh DBs create `venue_commands.snapshot_id NOT NULL`; legacy DBs are
  migrated with a nullable column but still use Python enforcement for new
  writes.
