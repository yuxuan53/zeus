## Command grammar mapping

M4 does not add `CommandState` or `CommandEventType` values. Typed cancel outcomes map to existing command grammar:

| Typed M4 outcome | Existing command event | Existing command state | Replacement behavior |
|---|---|---|---|
| `CANCELED` | `CANCEL_ACKED` | `CANCELLED` | replacement may proceed after terminal release |
| `NOT_CANCELED` | `CANCEL_FAILED` | `REVIEW_REQUIRED` | replacement blocked until operator/reconcile action |
| `UNKNOWN` | `CANCEL_REPLACE_BLOCKED` | `REVIEW_REQUIRED` | replacement blocked until M5 proves absence |

The U2 order-fact vocabulary may still use semantic labels such as `CANCEL_CONFIRMED`, `CANCEL_FAILED`, and `CANCEL_UNKNOWN`; these are not command-state grammar.

## Mutex schema

`exit_mutex_holdings` is keyed by `f"{position_id}:{token_id}"`. Active rows have `released_at IS NULL`. Terminal command transitions release via `src/state/venue_command_repo.append_event()` after collateral release.

`ExitMutex.acquire()` is fail-closed for released-row reuse: it reacquires only with a conditional compare-and-swap against the observed `(mutex_key, command_id, released_at)` row. If another writer acquired the key between read and write, the stale acquire returns `False` unless the same command is already the active holder.
