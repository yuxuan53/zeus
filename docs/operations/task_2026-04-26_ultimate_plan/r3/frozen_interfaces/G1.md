# G1 interface draft — live-readiness gates

Status: DRAFT / NOT CLOSED
Date: 2026-04-27

This file records the intended G1 interface during implementation. It is not a
closed frozen interface until pre-close critic and verifier approval are
recorded and `_phase_status.yaml` is updated to `COMPLETE`.

## Interface

- Entry point: `python3 scripts/live_readiness_check.py [--json] [--evidence-root PATH]`
- Gate count: exactly 17 readiness gates, IDs `G1-01` through `G1-17`.
- Output:
  - plain text summary by default
  - machine-readable JSON with per-gate `PASS`/`FAIL` evidence under `--json`
- Exit code:
  - `0` only when all 17 gates pass and staged-live-smoke evidence passes
  - `1` when any gate fails or staged-live-smoke evidence is missing/invalid

## Non-authority

- `live_deploy_authorized` remains `false`.
- Operator gate `live-money-deploy-go` remains required.
- The script does not run live smoke, submit, cancel, redeem, cut over, activate
  credentials, promote strategies, or mutate production DB/state.
