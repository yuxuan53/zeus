# Z2 Frozen Interface — PolymarketV2Adapter

Status: frozen after Z2 pre-close critic+verifier approval on 2026-04-27.
This file freezes the public adapter contract for downstream R3 phases.

## Public classes

- `src/venue/polymarket_v2_adapter.py::PolymarketV2Adapter`
- `src/contracts/venue_submission_envelope.py::VenueSubmissionEnvelope`

## Stable semantics

- Provenance is pinned at `VenueSubmissionEnvelope`, not at one SDK method.
- `preflight()` fails closed when Q1-zeus-egress evidence is absent.
- `submit(envelope)` and `submit_limit_order(...)` call `preflight()` before
  SDK contact and return typed rejections without SDK calls when Q1 evidence is
  absent.
- `submit(envelope)` never reports success without an order id.
- `PolymarketClient.place_limit_order()` is a compatibility wrapper only; it
  calls adapter `preflight()` and returns a rejected dict before any submit when
  the adapter preflight fails.
- `VenueSubmissionEnvelope.outcome_label` is restricted to `YES` or `NO`.
- Fresh executable snapshots must provide either an explicit `is_fresh` signal
  or `captured_at + freshness_window_seconds`; stale timestamped snapshots are
  rejected before envelope creation.
- Legacy compatibility envelopes are placeholders until U1 lands executable
  market snapshots; their canonical hashes are still computed from final
  side/price/size before SDK contact.
- SDK-specific request/response data must be hashed or serialized into the
  envelope/result surfaces without storing secrets.

## Forbidden downstream assumptions

- Do not assume a one-step-only or two-step-only SDK shape.
- Do not mutate `venue_commands` schema from adapter code.
- Do not treat this interface as proof that live cutover is approved.
- Do not use the legacy compatibility placeholder as U1-certified market
  identity or source truth.
