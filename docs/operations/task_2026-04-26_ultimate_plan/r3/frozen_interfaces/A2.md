## Guarantees

- `RiskAllocator.can_allocate()` returns structured allow/deny decisions for per-market, per-event, per-resolution-window, correlated exposure, reduce-only, same-market unknown side effects, kill switch, drawdown, heartbeat, WS-gap, and reconciliation thresholds.
- OPTIMISTIC and CONFIRMED exposure are distinct: confirmed exposure counts at full weight; optimistic exposure counts at `CapPolicy.optimistic_exposure_weight`.
- `load_position_lots()` is read-only and consumes latest append-only `position_lots` state per position; it does not repair, insert, update, delete, or mutate production truth.
- Entry `SUBMIT_REQUESTED` command events persist allocation metadata so later lot reads can reconstruct event/window/correlation capacity from command truth.
- `PortfolioGovernor.update_state()` arms kill switch state for manual and automatic threshold breaches.
- `executor.py` consults the global allocator before entry command persistence or SDK contact and raises `AllocationDenied` on denial.
- `executor.py` consults the global submit kill-switch guard before exit command persistence or SDK contact. Reduce-only exits may proceed through reduce-only posture only when no kill-switch reason is active.
- `executor.py` persists and submits the allocator-selected order type: healthy/deep/far-from-close maker paths use `GTC`, while shallow/near-resolution/non-resting-only paths use `FOK`.
- `src/data/polymarket_client.py` preserves the selected order type when delegating to `PolymarketV2Adapter.submit_limit_order()`.
- `cycle_runner.py` refreshes the global allocator/governor from canonical read models at cycle start and surfaces `summary["portfolio_governor"]`.
