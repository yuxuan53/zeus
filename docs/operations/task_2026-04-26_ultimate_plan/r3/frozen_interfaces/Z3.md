# Frozen Interface — Z3 HeartbeatSupervisor

Date: 2026-04-27
Phase: Z3
Status: pre-close candidate

## Contract

- `src/control/heartbeat_supervisor.py` owns heartbeat health for venue resting-order placement.
- `HeartbeatHealth` states: `STARTING`, `HEALTHY`, `DEGRADED`, `LOST`, `DISABLED_FOR_NON_RESTING_ONLY`.
- `heartbeat_required_for(order_type)` returns false only for `FOK`/`FAK`; `GTC`, `GTD`, and unknown order types require HEALTHY heartbeat.
- `assert_heartbeat_allows_order_type("GTC")` raises `HeartbeatNotHealthy` unless global heartbeat status is `HEALTHY`.
- Executor live entry/exit paths call the heartbeat gate before venue-command persistence or SDK contact.
- Two consecutive heartbeat misses write `state/auto_pause_failclosed.tombstone` with reason `heartbeat_cancel_suspected`.
- Z3 reuses the existing fail-closed tombstone. It must not introduce another tombstone source.

## Deferred by design

- M5 owns exchange reconciliation findings and local order classification after heartbeat loss.
- T1 owns fake venue forced heartbeat-miss integration.
- Z4/R1 own pUSD, collateral, and redemption/FX behavior.
